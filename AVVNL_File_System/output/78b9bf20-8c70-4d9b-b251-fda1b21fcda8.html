Difference between c & C++:-
•C is a procedural oriented programming language (POP) whereas C++ is a object oriented language(OOPS).
•C has top-down approach & C++ has bottom-up approach.
•In C the variable & functions must be declared in the beginning of the function, while in C++ the declaration may anywhere in the function before its use.
•C++ provides the reusability of code rather than writing the code from the beginning.
•In C++ it provides data hiding with data security but not in C language.
•C++ provides the S/W complexity that can be easily maintain in C++ but not in C.
Basic Concepts of Object Oriented Programming(OOPS):-
1.Objects: -
Objects are the basic runtime entity in an object oriented system. They may represent a person, A place, a bank A/c, a table of data or any item that the program has to handle. In object oriented programming, an object is a instance variable of class by which we can access the property of that class. Each object contains data & code to manipulate the data. Object can interact without having to know the details of each other data or code.
2.Classes: -
A class is a collection of data members & member functions. In other word, a class is a user defined data type in which, we can define any number of variable & function.
We just mention that object contain data & code to manipulate the data, the entire set of data & code of an object can be made a user defined data type with the help of a class.
Infect, objects are variable of the type class. Once a class has been defined, we can create any number of objects belonging to that class. A class is thus a collection of objects of similar types.
3.Data Abstraction & Encapsulation: -
The wrapping up of data & functions into a single unit is known as encapsulation. It is the most striking feature of a class. The data is not accessible to the outside world & only those functions which are wrapped in the class can access that data. These functions provide the interface between the object’s data and the program. This insulation of data from direct access by the program is called data hiding or information hiding.
Abstraction refers to the act of representing essential features without including the background detail. Classes used the concept of abstraction & are defined as a list of abstract attribute such as size, weight & cost and functions to operate on the attributes. They encapsulate all the essential properties of the object that are to be created. The attributes are sometimes called data members because they hold the information & the function that operate on these data are sometimes called methods or member functions.
4.Inheritance: -
Inheritance is the process by which objects of one class acquire the properties of objects of another class. It supports the concept of Hierarchical Classification (A tree like structure) for eg. Muzaffarnagar is the part of UP which is again is the part of class “Area”. The principle behind this sort of division is that each derived class share common properties / characteristics with the class from which it is derived.
In OOPS concept of inheritance provides the idea of Reusability. This means that we can add additional features to an existing class without modifying it.
